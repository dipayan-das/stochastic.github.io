• The Maze solving Grabber Robot   
  • Under guidance of Prof. Nabanita DasGupta
    Project Partners - Debajit De, Subhra Kamal Sengupta, Arnab Sarkar
The goal of our project was to develop a fully automatic system, which will be able to grab object and transport it from its start location to end location through maze.The main challenge was to navigate through mazes all by itself, by having a knowledge of the maze or by solving the maze 1st time with much amount of time, and other major challenge was to differentiate between a moveable object and a wall.Details.The description is outdated but still gives some idea about the working and building of the robot.The robot has gone through few upgrades after that.

• Twenty-Nine                                     
    Project Partners - Dibyendu Das (Tufts University, Graduate Student)
The aim of this project is to create 29 (a card game very popular in eastern Asia Wiki).and played with 4 player, and the challenge is to make the Computer Player recognize the human players playing attribute and respond to it accordingly.

The computer player's bidding decision is dependent on his hand set which is deterministic,and his playing attributes, his partner's bidding amount which are stochastic.Our goal is to make the bidding of the computer player realistic with the implementation of probabilistic modelling. The game is written on python

The project is currently under testing. Source Code
